Introdução geral

Introdução geral

O Intel powered classmate PC Theft Deterrent Agent é executado no Intel powered classmate PC de um aluno, que é projetado levando-se em especial consideração as questões de segurança. O Intel powered classmate PC Theft Deterrent Agent ac eita inglês, espanhol e português.

 

O software Intel powered classmate PC Theft Deterrent Agent oferece uma interface do usuário simples, que pode realizar as seguintes funções:

  • Configurar diferentes opções de rede para diferentes situações.

  • Exibir o status atual do certificado e informações sobre o dispositivo.

  • Verificar o certificado de autorização sempre que o computador for iniciado.

  • Lembrar o proprietário de obter um novo certificado de autorização, caso o certificado atual esteja prestes a expirar ou já tenha expirado.

 

O proprietário do Classmate PC pode obter um certificado de autorização periodicamente a partir do Intel powered classmate PC Theft Deterrent Server. O certificado de autorização é o único passaporte para o Intel powered classmate PC. O software Intel powered classmate PC Theft Deterrent Agent verifica esse certificado de autorização e determina por quanto tempo o Intel powered classmate PC pode ser utilizado.

 

Se um dispositivo for roubado, o Intel powered classmate PC Theft Deterrent Server gerará uma solicitação de certificado de inicialização individual, com “hoje” como a Data de expiração e “0” como o Certificado de inicialização. Uma vez que o Intel powered classmate PC receba esse certificado, ele não funcionará mais após a reinicialização do sistema. Esse recurso tem como objetivo reduzir o risco de roubo dos Intel powered classmate PCs.

Utilizando o Intel powered classmate PC Theft Deterrent Agent

Ícone na bandeja do sistema

O programa Intel powered classmate PC Theft Deterrent Agent será iniciado automaticamente quando o sistema operacional for iniciado. O programa será minimizado, exibindo um ícone na bandeja do sistema do Windows. A cor do ícone muda para indicar um status diferente. Quando o usuário move o mouse sobre o ícone, ele exibirá uma mensagem de dica de contexto.

 Status “Normal” A dica de contexto exibirá a mensagem “Normal”.

  Status “Aviso” Esse status indica que o computador tem menos de 2 dias ou menos de 5 vezes para expirar. A dica de contexto exibirá a mensagem “Aviso”. O Theft Deterrent Agent fará download do certificado de inicialização comum automaticamente.

  Status “Desativado” Esse status indica que a conta desse computador não foi aprovada no theft deterrent server. A dica de contexto exibirá a mensagem “Desativado”.

  Status “Não é possível efetuar a conexão com o servidor” Isso indica que o computador não pode fazer conexão com o servidor. A dica de contexto exibirá a mensagem “Não é possível efetuar a conexão com o servidor”. Verifique a rede para fazer a conexão do Theft Deterrent Agent com o Theft Deterrent Server.

  Os status “Não é possível efetuar a conexão com o servidor” e “Desativado” indicam que o computador não pode efetuar conexão com o servidor e que a conta não foi aprovada. A dica de contexto exibirá a mensagem “Não é possível efetuar a conexão com o servidor”.

Informar código de desbloqueio

Se a data do sistema for posterior à data de expiração ou se o contador de inicialização for zerado, o BIOS do computador não carregará o sistema operacional. Se o sistema não passar pela verificação anterior à inicialização, o BIOS dará ao usuário a oportunidade de inserir um código de desbloqueio, conforme indicado a seguir. O usuário pode obter esse código na sua página de login. Consulte o Capítulo 2.2.2.

 

Após obter um “código de desbloqueio”, o usuário deve responder as seguintes informações e orientações fornecidas na tela do BIOS:

  1. “O certificado de inicialização do computador já está vencido e a máquina está bloqueada no momento.”

  2. “Entre em contato com o administrador do seu computador para obter um código de desbloqueio para o seu computador ser desbloqueado.”

  3. “ID do hardware”

  4.  “Escala de inicializações”:

  5.  “Está pronto para inserir o código de desbloqueio?” (Pressione S para Sim ou N para Não. O sistema fechará se você digitar N.)

  6. “Insira um número de 10 dígitos.”

 

Se o código for válido, o BIOS reservará um prazo de 2 dias e 5 inicializações para permitir que o usuário atualize o certificado de inicialização imediatamente. Em seguida, o BIOS carregará o sistema operacional. Se o código for inválido, o BIOS exibirá as seguintes informações:

  1. O código de desbloqueio inserido está errado! Tente novamente.

  2. O código de desbloqueio foi digitado incorretamente três vezes O sistema fechará imediatamente.

Analise o certificado em disco USB

Se o computador estiver bloqueado, os usuários podem obter certificados no disco USB do Administrador de TI. Em seguida, eles inserem o disco USB no computador e pressionam uma tecla de atalho especial (Ctrl + Insert). O BIOS pode analisar o certificado automaticamente.


Perguntas freqüentes

  1. Se o endereço do servidor foi alterado, devo alterar o endereço do servidor manualmente?

    2. R: Não é necessário. O Intel powered classmate PC Theft Deterrent Agent pode pesquisar o endereço do servidor, automaticamente, quando ele descobre que o endereço do servidor não está correto.

     

  2. Por que o Agent não oferece um atalho para inicialização e um método de encerramento?

    3. R: Sendo um programa de gerenciamento responsável pela manutenção do estado do sistema, o Intel powered classmate PC Theft Deterrent Agent deve ser mantido em execução o tempo todo. Ele não deve ser encerrado pelos usuários.

     

  3. Se o Intel powered classmate PC Theft Deterrent Agent for encerrado, a função theft-deterrent ainda será válida se o PC for roubado?

    4. R: Sim. Mesmo que o programa Intel powered classmate PC Theft Deterrent Agent não esteja sendo executado, o BIOS do computador continuará não carregando o sistema operacional após a expiração. Além disso, na próxima vez em que o computador for iniciado, o programa Intel powered classmate PC Theft Deterrent Agent será carregado automaticamente na inicialização do sistema.

     

  4. Posso desinstalar o programa Intel powered classmate PC Theft Deterrent Agent? E, caso isso seja possível, o que ocorrerá se eu desinstalar o programa Intel powered classmate PC Theft Deterrent Agent?

    5. R: Sendo um programa de gerenciamento, o Intel powered classmate PC Theft Deterrent Agent deve ser mantido em execução o tempo todo. Entretanto, você pode desinstalar Intel powered classmate PC Theft Deterrent Agent executando o arquivo de script de desinstalação no pacote de desinstalação, que se pode obter com o administrador.

     

  5. O Classmate PC não inicia, embora apenas uma das condições de expiração esteja presente (por exemplo, o número máximo de inicializações ainda não foi atingido, mas a data atual ultrapassou a data de expiração).

    6. R: Quando uma das duas condições de expiração (data ou número de inicializações) ocorrer, o Classmate PC não será iniciado.

     

  6. O status de aviso afetará as funções do computador (por exemplo, algumas funções não poderão ser desempenhadas)?

    7. R: Não. O status de aviso apenas informa ao usuário para fazer download de um novo certificado de inicialização para manter o uso regular do computador.

     

  7. O que é um código de desbloqueio?

    8. R: O código de desbloqueio permite que o Classmate PC seja utilizado até 31/12/2099, por dois dias e num total de 10 inicializações, mesmo que um novo certificado de inicialização não tenha sido obtido a partir do servidor. Nesses dois dias, os usuários devem obter o novo certificado, fazendo download a partir do servidor configurado.

     

  8. É possível estender o tempo de utilização do computador, modificando a hora/data do sistema do computador?

    9. R: Sim. Alterar a hora/data do sistema pode estender o tempo de utilização do computador. No entanto, alterar a data/hora do sistema impedirá o Intel powered classmate PC Theft Deterrent Agent de fazer download de um novo certificado de inicialização a partir do Intel powered classmate PC Theft Deterrent Server. Como o número permitido de inicializações do sistema até a expiração não é afetado pela alteração da hora/data, o certificado irá, conseqüentemente, expirar quando o número de inicializações permitido for alcançado, mesmo que a data de expiração nunca seja alcançada.

     

  9. Por que não posso visitar o Theft Deterrent Server no Firefox após o servidor ser reinstalado? O Firefox envia uma caixa de mensagens pop-up que mostra que o certificado é inválido.

    10. R: Se você selecionar “Aceitar certificado do servidor permanentemente” quando visitar o Theft Deterrent Server no Firefox, você não receberá mais o aviso de aceitação do certificado novamente. Entretanto, assim que o Theft Deterrent Server for instalado novamente, você deve excluir manualmente o certificado anterior do Firefox. Caso contrário, você não poderá visitar o Theft Deterrent Server com êxito usando SSL, pois o certificado do servidor anterior entra em conflito com o novo.
